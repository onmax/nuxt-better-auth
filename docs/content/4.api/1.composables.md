## useUserSession

```ts
const session = useUserSession()
```

### Return Values

| Property | Type | Description |
| :--- | :--- | :--- |
| `loggedIn` | `ComputedRef<boolean>` | Returns `true` if the user is currently authenticated. |
| `user` | `Ref<AuthUser | null>` | The current user object. Type is automatically inferred. |
| `session` | `Ref<AuthSession | null>` | The current session object. |
| `ready` | `Ref<boolean>` | `true` when the initial session fetch is complete. |
| `client` | `AuthClient | null` | Direct access to the Better Auth client instance. |

### Methods

#### `signIn`

Proxies the Better Auth `signIn` method. It waits for the local session state to update before resolving, ensuring `loggedIn` is true immediately after the await.

```ts
await signIn.email({
  email: 'user@example.com',
  password: 'password'
})
```

#### `signOut`

Signs the user out and clears the local session state.

```ts
await signOut()
```

#### `fetchSession`

Manually triggers a session refresh. Useful if you've updated user data on the server via a side channel.

```ts
await fetchSession()
```

#### `updateUser`

Optimistically updates the local user object.

```ts
updateUser({ name: 'New Name' })
```

::callout{icon="i-lucide-zap"}
**Reactivity**: `user` and `session` are global states using `useState`. Changes in one component are instantly reflected everywhere.
