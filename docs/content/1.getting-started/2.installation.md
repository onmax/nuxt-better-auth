## Prerequisites

- Nuxt v3.10+
- [NuxtHub](https://hub.nuxt.com) v0.10+

::callout{icon="i-lucide-info" to="/guides/database-less-mode"}
Database is optional. See **Database-less Mode** for OAuth-only and edge deployments.
::

## Add to project

::steps

### Install the module

```bash
npx nuxi module add @onmax/nuxt-better-auth
```

::callout{icon="i-lucide-triangle-alert" color="warning"}
**pnpm users**: Add `auto-install-peers=true` to `.npmrc` or install manually: `pnpm add better-auth`
::

### Set Environment Variables

Create a `.env` file in the root of your project and add the following environment variables:

1. **Secret Key**

A secret value used for encryption and hashing. It must be at least 32 characters and generated with high entropy. Click the button below to generate one. You can also use `openssl rand -base64 32` to generate one.

```txt [.env]
NUXT_BETTER_AUTH_SECRET=
```

::generate-secret
::

2. **Base URL** (production only)

```txt [.env]
NUXT_PUBLIC_SITE_URL=https://your-domain.com
```

### Configure NuxtHub

Ensure `@nuxthub/core` is present and configured with a database.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: [
    '@nuxthub/core',
    '@onmax/nuxt-better-auth'
  ],

  hub: {
    db: 'sqlite' // 'sqlite' | 'postgresql' | 'mysql'
  }
})
```

### Create Configuration Files

The module requires two configuration files to link Nuxt with Better Auth.

1. **Server Configuration**: `server/auth.config.ts`
2. **Client Configuration**: `app/auth.client.ts`

::

Continue to **Configuration** to set up these files.
