---
title: Database-less Mode
description: Run Better Auth without a database for edge and serverless deployments.
---

## Overview

The module auto-detects when no `hub.db` is configured and runs in **stateless mode**:
- Sessions stored in encrypted cookies (JWE)
- OAuth state in cookies
- No schema generation

## Configuration

Simply omit `hub.db` from your config:

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  hub: { kv: true }, // optional: KV for session storage
  // No hub.db = database-less mode
})
```

## With KV Secondary Storage

For better session management, enable KV:

```ts
export default defineNuxtConfig({
  hub: { kv: true },
  auth: { secondaryStorage: true },
})
```

This stores sessions in KV with TTL, enabling session listing and revocation.

## Limitations

Without a database:
- **Email/password auth** unavailable (no account storage)
- **Session listing** limited to KV/cookies
- **User profile updates** not persisted

Best for: OAuth-only flows, read-heavy apps, edge deployments.

## How It Works

When no database is configured, Better Auth automatically enables:
- `session.cookieCache.enabled: true`
- `session.cookieCache.strategy: 'jwe'`
- `account.storeStateStrategy: 'cookie'`
