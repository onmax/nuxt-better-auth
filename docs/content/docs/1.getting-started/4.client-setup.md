title: Client Setup
description: Configure the client-side authentication client.
---

To enable client-side features and plugins, you must export a client factory function from `app/auth.client.ts`. This allows the module to instantiate the Better Auth client with the correct `baseURL` and matching plugins.

## Create the Client Factory

Create a file at `app/auth.client.ts` (or `.js`) and export a `createAppAuthClient` function.

```ts [app/auth.client.ts]
import { createAuthClient } from 'better-auth/vue'

export function createAppAuthClient(baseURL: string) {
  return createAuthClient({
    baseURL,
    // Add client-side plugins here
    plugins: []
  })
}

// Export type for auto-inference
export type AppAuthClient = ReturnType<typeof createAppAuthClient>
```

::callout{icon="i-lucide-info"}
This file is required because Better Auth plugins often have a client-side counterpart that must be registered here to work (e.g., `adminClient`, `twoFactorClient`).
::

## Using Plugins

If you added a plugin in `server/auth.config.ts`, make sure to add its client equivalent here.

```ts [app/auth.client.ts]
import { createAuthClient } from 'better-auth/vue'
import { adminClient } from 'better-auth/client/plugins'

export function createAppAuthClient(baseURL: string) {
  return createAuthClient({
    baseURL,
    plugins: [
      adminClient() // Must match the server plugin
    ]
  })
}
```
